# Purpose

This document establishes the design principles governing every interface within The Garden.

The Observatory should never feel like a collection of unrelated menus.

Every panel, visualization, and interaction should express one coherent design philosophy focused on understanding a living world.

---

# Core Principle

**Interfaces should reduce complexity without hiding it.**

The Garden contains extraordinary simulation depth.

The role of the interface is not to simplify the world.

It is to make complexity understandable.

---

# Design Philosophy

Every interface should help players answer questions naturally.

Interfaces exist to support investigation.

They should never distract from the simulation itself.

Good interface design becomes nearly invisible.

---

# Principle 1 — Clarity Before Density

Present only the information needed for the current level of investigation.

Important information should be immediately recognizable.

Secondary information should remain available without competing for attention.

---

# Principle 2 — Progressive Disclosure

Information should unfold naturally.

Overview first.

Details second.

Evidence third.

Players choose how deeply to investigate.

No interface should expose every available statistic immediately.

---

# Principle 3 — Context Matters

Information should always appear within meaningful context.

A citizen is understood through:

Family.

Occupation.

Community.

Civilization.

History.

Relationships.

No entity should be presented in isolation.

---

# Principle 4 — Consistency

Similar interactions should behave consistently throughout the Observatory.

Navigation.

Search.

Filtering.

Sorting.

Comparison.

History.

Bookmarks.

Annotations.

Consistency reduces learning effort.

---

# Principle 5 — Readability

Interfaces should prioritize readability over decoration.

Typography.

Spacing.

Grouping.

Contrast.

Alignment.

Visual hierarchy.

Every design decision should improve comprehension.

---

# Principle 6 — Calm Design

The Observatory should feel calm.

Avoid:

Visual noise.

Animation overload.

Excessive color.

Constant movement.

Persistent alerts.

The interface should encourage thoughtful observation.

---

# Principle 7 — Discoverability

Powerful tools should be easy to discover.

Advanced functionality should emerge naturally through exploration rather than requiring extensive documentation.

Learning should feel intuitive.

---

# Principle 8 — Connected Information

Every piece of information should connect to related knowledge.

Examples include:

Citizen → Family

Family → Settlement

Settlement → Civilization

Civilization → Timeline

Timeline → Historical Event

Historical Event → Environmental Change

Knowledge forms an interconnected network.

---

# Principle 9 — Explainability

Whenever possible, conclusions should be traceable.

If the Observatory reports:

Population declined.

The player should be able to investigate why.

Interfaces should encourage evidence-based understanding.

---

# Principle 10 — Player Choice

Players choose how they interact.

Some prefer quick summaries.

Others prefer extensive analysis.

The Observatory should support both without favoring either.

---

# Information Hierarchy

Every interface should present information in layers.

Level 1

Immediate overview.

Level 2

Important details.

Level 3

Supporting evidence.

Level 4

Historical context.

Level 5

Cross-system relationships.

This hierarchy prevents information overload while preserving depth.

---

# Error Prevention

Interfaces should minimize accidental actions.

Important interventions should be:

Clear.

Reversible where appropriate.

Well explained.

Intentional.

The player should rarely feel punished by the interface itself.

---

# State Variables

The Observatory presents:

Information Priority.

Context Depth.

Navigation State.

Investigation Depth.

Visualization Preferences.

Search History.

Bookmarks.

Annotations.

These values personalize presentation without affecting the simulation.

Interface Principles guide:

History Explorer.

Accessibility.

Every Observatory feature should follow these principles.

Future Observatory modules.

# Closing Statement

A scholar entered a vast library.

Millions of books surrounded them.

Yet finding knowledge did not require reading every page.

Clear shelves guided exploration.

Thoughtful organization revealed relationships.

Each answer pointed naturally toward another question.

The library never became smaller.

It simply became understandable.

The Observatory should do the same for an entire world.
