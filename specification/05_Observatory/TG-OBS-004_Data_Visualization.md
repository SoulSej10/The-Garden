# Purpose

This document defines how simulation data is transformed into meaningful visual information.

The Observatory should not present raw statistics as an end in themselves.

Instead, it should reveal relationships, trends, patterns, and consequences that help players understand how the living world changes over time.

---

# Core Principle

**Visualization should explain, not merely display.**

Every graph, chart, map, and indicator should answer a meaningful question about the simulation.

If a visualization does not improve understanding, it should not exist.

---

# Design Philosophy

The Garden generates extraordinary amounts of information.

Players should never be expected to interpret thousands of numbers.

Instead, the Observatory presents carefully designed visualizations that reveal:

Patterns.

Relationships.

Trends.

Anomalies.

Change over time.

Cause and effect.

The visualization is a lens, not a database.

---

# Layers of Understanding

Every visualization should support multiple levels of exploration.

Level 1 — Summary

A quick overview of the current state.

Level 2 — Trends

How values have changed over time.

Level 3 — Relationships

How different systems influence one another.

Level 4 — Evidence

The underlying events and observations that produced the visualization.

Players choose how deeply to investigate.

---

# Types of Visualization

The Observatory may present information through:

Line graphs.

Bar charts.

Heat maps.

Flow diagrams.

Population pyramids.

Tree structures.

Relationship networks.

Timelines.

Geographic overlays.

Comparative dashboards.

Animations illustrating change over time.

The chosen format should always match the nature of the information.

---

# Temporal Visualization

Time is one of the Observatory's most important dimensions.

Visualizations should clearly communicate:

Growth.

Decline.

Cycles.

Seasonality.

Historical change.

Long-term trends.

Players should understand not only *what* happened, but *when* and *how quickly* it happened.

---

# Spatial Visualization

Data should remain connected to geography.

Examples include:

Population density.

Forest coverage.

Trade routes.

Migration paths.

Rainfall distribution.

Disease spread.

Infrastructure development.

Environmental recovery.

Spatial visualization connects abstract information to the physical world.

---

# Comparative Analysis

Players should be able to compare:

Different civilizations.

Different ecosystems.

Different regions.

Different historical periods.

Different citizens.

Different policies.

Comparisons should highlight meaningful differences without implying that one outcome is universally superior.

---

# Causal Visualization

Whenever possible, visualizations should reveal causal relationships.

For example:

Deforestation

↓

Soil degradation

↓

Reduced harvests

↓

Migration

↓

Urban growth

The Observatory should make interconnected systems understandable without oversimplifying them.

---

# Visual Simplicity

Visualizations should remain:

Readable.

Consistent.

Minimal.

Accessible.

Contextual.

Decorative complexity should never obscure understanding.

---

# Interactive Exploration

Players should be able to:

Filter.

Zoom.

Compare.

Highlight.

Trace relationships.

Navigate through time.

Bookmark findings.

Every interaction should support investigation rather than increase interface complexity.

---

# Scientific Integrity

Visualizations should represent the simulation honestly.

They should never exaggerate differences through misleading scales or selective presentation.

Uncertainty should remain visible whenever certainty does not exist.

The Observatory values truthful representation over dramatic presentation.

---

# State Variables

The Observatory presents:

Visualization Type.

Observed Variables.

Time Range.

Spatial Scope.

Comparison Targets.

Filtering Rules.

Highlight Selection.

Investigation Depth.

These values control presentation while leaving the simulation unchanged.

# Closing Statement

A player opened a population graph.

At first, it appeared to be a simple rise and fall.

They expanded the timeline.

A drought became visible.

A migration followed.

Trade routes shifted.

A neighboring city expanded.

Forest regeneration restored the watershed.

Population recovered.

The graph had never been about numbers.

It was the visual history of thousands of individual lives.

In The Garden, every chart tells a story waiting to be understood.
